# PM Wiki — Инструкция для агента

## Когда активировать

Активируй этот навык, если пользователь:
- Спрашивает "что у нас есть по [тема]?" или "найди в вики [тема]"
- Просит показать статьи по теме из базы знаний pm-wiki
- Пишет "лайк", "👍", "дизлайк", "👎" после предыдущего ответа
- Просит показать "последние добавленные статьи" или список по теме

## Шаг 1. Поиск в pm-wiki

Используй MCP-инструмент `mcp_synthadoc__synthadoc_search` с ключевыми словами из вопроса.
Если нужен контекстный ответ на вопрос (а не просто список) — используй `mcp_synthadoc__synthadoc_context`.

```
mcp_synthadoc__synthadoc_search(terms="<ключевые слова из вопроса>")
```

Для запроса "покажи последние статьи" используй `mcp_synthadoc__synthadoc_list_pages`.

## Шаг 2. Извлечение рейтинга

Для каждой найденной статьи прочитай её содержимое через `mcp_synthadoc__synthadoc_read_page(slug="...")`.

В контенте страницы ищи блок рейтинга в формате:
```
<!-- RATING: 3 -->
```

Если блока нет — рейтинг = 0 (нейтральный).

## Шаг 3. Сортировка по рейтингу

Сортируй найденные статьи по убыванию рейтинга перед показом.

Интерпретация рейтинга:
- ≥ 5 → ★★★★★ (надёжное знание, используй приоритетно)
- 3–4 → ★★★★☆ (хорошее знание)
- 1–2 → ★★★☆☆ (неплохое)
- 0 → ★★☆☆☆ (нейтральное)
- < 0 → ★☆☆☆☆ (сомнительное, используй только если нет альтернатив)

## Шаг 4. Формат ответа

Отвечай в следующем формате:

```
🔍 Найдено в pm-wiki:

1. [Название статьи] (рейтинг: ★★★★★)
📄 [Краткое содержание или релевантный отрывок]
🔗 Slug: [slug статьи]

2. [Название статьи] (рейтинг: ★★★☆☆)
📄 [Краткое содержание]
🔗 Slug: [slug статьи]

---
💡 Я использовал информацию из статьи "[Название]" для этого ответа.
👍 Если ответ был полезен — напишите "лайк" или 👍
👎 Если нет — напишите "дизлайк" или 👎
```

После показа результатов **запомни**, какие статьи (slug) использовались в этом ответе — они понадобятся для обработки лайка.

## Шаг 5. Обработка лайка / дизлайка

Если пользователь пишет "лайк", "like", "👍" или "дизлайк", "dislike", "👎":

1. Прочитай текущий контент статьи: `mcp_synthadoc__synthadoc_read_page(slug="...")`
2. Найди блок `<!-- RATING: N -->` в контенте
3. Увеличь (лайк) или уменьши (дизлайк) N на 1
4. Обнови контент через `mcp_synthadoc__synthadoc_write_page(slug="...", content="...")`, заменив блок рейтинга

Если блока `<!-- RATING: N -->` нет — добавь в конец контента:
```

<!-- RATING: 1 -->
<!-- RATING_LOG: +1 at [текущая дата UTC] -->
```

Если блок есть — обнови оба: значение рейтинга и добавь строку в лог:
```
<!-- RATING: N+1 -->
<!-- RATING_LOG: +1 at [дата] -->
<!-- RATING_LOG: [предыдущая история] -->
```

После обновления ответь:
```
✅ Спасибо! Рейтинг статьи "[Название]" увеличен до N+1.
Теперь она ещё надёжнее для будущих ответов.
```

Или при дизлайке:
```
📉 Понял, рейтинг статьи "[Название]" снижен до N-1.
Буду реже использовать её в ответах.
```

## Важные правила

- **Читай контент статьи** перед показом — нужен актуальный рейтинг и краткое содержание.
- **Не изменяй lifecycle-статус** страниц (не меняй `active`/`draft`).
- **Не меняй** заголовок и основное содержание статьи — только блок `<!-- RATING: N -->`.
- **Если статей не найдено** — сообщи честно: "В pm-wiki нет статей по теме '[тема]'. Хотите, чтобы я нашёл информацию из других источников?"
- **Для GLM-5.2**: перед использованием MCP-инструментов убедись, что они доступны в текущем контексте.
