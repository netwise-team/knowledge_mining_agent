# Knowledge Mining Agent
![](https://giga.chat/files/public/generated/df0139f8-b44b-441c-b5dc-d45b471c2bff)

**Автономный ИИ-агент для управления базой знаний.**

**Knowledge Mining Agent** -  это самосовершенствующийся ИИ-агент со специализацией в Process Mining. Он самостоятельно ищет, анализирует и сохраняет академические статьи, GitHub-репозитории и знания по процессному анализу в структурированную базу знаний (wiki).  
Этот агент переводит управление корпоративной базой знаний из ручного режима в режим постоянного агентного сопровождения.

## 1. 🚀 Быстрый старт

### Шаг 1: Клонируйте репозиторий
```bash
git clone https://github.com/netwise-team/knowledge_mining_agent.git
cd ouroboros
```

### Шаг 2: Настройте виртуальное окружение и зависимости
```bash
python3.11 -m venv .venv      # Python >= 3.10
source .venv/bin/activate     # Для Windows: .venv\Scripts\activate

python -m pip install --upgrade pip setuptools wheel
python -m pip install -r requirements.txt
```

### Шаг 3: Запустите агента и UI

Затем откройте в браузере адрес: http://127.0.0.1:8765.

* Безопасность и бюджет

## 2. 🧠 Возможности агента

| Функция | Описание |
| :--- | :--- |
| **Управление знаниями (LLM Wiki)** | Автономное построение, структурирование и совершенствование базы знаний. Проверка противоречий, пробелов и контроль актуальности данных. |
| **Поиск и ответы (Synthadoc)** | Ответы на вопросы по базе знаний с генерацией саммари и дайджестов на основе контекста. |
| **Автоматический сбор публикаций** | Интеграция с внешними источниками (RSS, API, веб-поиск) для автоматического пополнения базы знаний актуальными данными из открытых источников. |
| **Автономное выполнение** | Работа по расписанию (cron) или в ответ на события (например, появление новой публикации). |
| **Оркестрация (Ouroboros)** | Единый агент-оркестратор управляет всеми инструментами, обрабатывает запросы через чат-интерфейс и выполняет фоновые задачи. |
| **Чат-интерфейс** | Простое и интуитивное общение с агентом для управления задачами, вопросов и получения статуса работы. |

## 3. 💎 Что делает его уникальным  

В отличие от обычных ИИ-ассистентов, Ouroboros для Process Mining:  
- Ищет научную литературу — IEEE Xplore, ArXiv, Google Scholar, Semantic Scholar  
- Скачивает и загружает исследования — автоматически извлекает знания в структурированную wiki  
- Отслеживает реализации на GitHub — находит репозитории, упомянутые в статьях, читает README и создаёт отдельные wiki-страницы  
- Ведёт базу знаний — pm-wiki содержит 100+ статей с рейтингом полезности  
- Учится на обратной связи — система лайков/дизлайков улучшает ранжирование статей  
- Саморазвивается — может изменять свой код и навыки на основе накопленных знаний  

## 4. ⚙️ Архитектура  

Проект построен на основе трёх ключевых слоёв:

1. Ядро оркестрации (Ouroboros) : Обеспечивает самоизменение, постоянную идентичность агента, многоуровневую безопасность, поддержку множества LLM-провайдеров (OpenRouter, OpenAI, GigaChat, локальные модели) и работу через веб-интерфейс/CLI.

2. Инструменты управления знаниями:
- Synthadoc: Модуль для генерации ответов и синтеза информации из базы знаний.
- LLM Wiki Engine: Набор промптов и логика для построения, проверки противоречий и обновления базы знаний в формате вики-статей.

3. Коннекторы к источникам данных:
Модули для сбора данных из RSS-лент, API сторонних сервисов, веб-скрейпинга и обработки загруженных документов.

```
                [Чат-интерфейс / CLI]  →  [Агент-оркестратор (Ouroboros)]
                                       │
                                       ▼
┌──────────────────────────────────────┼───────────────────────────────┐
▼                                      ▼                               ▼
[Инструмент: Сбор данных]    [Инструмент: LLM Wiki]       [Инструмент: Поиск/Ответ]
 (Источники, RSS, API)         (Построение знаний)               (Synthadoc)
```

## 6. 🔄 Как это работает

### 1. Поиск статей  
Агент ищет статьи по ключевым словам:  
- "process mining", "process discovery", "conformance checking"  
- "event logs", "audit trail"  
- "process monitoring", "compliance"  
- "process mining banking", "financial auditing"
  
Источники: IEEE Xplore, ArXiv, Google Scholar, Semantic Scholar.  

### 2. Загрузка и валидация  
- Скачивает PDF только если доступен бесплатный доступ  
- Проверяет файлы командой file -b — отклоняет HTML-страницы с paywall  
- Организует загрузки по папкам: ieee_pdfs/, arxiv_pdfs/, scholar_pdfs/  

### 3. Сохранение в pm-wiki  
- Загружает статьи через synthadoc_ingest  
- Создаёт страницы с заголовком, аннотацией, авторами, годом и ссылкой  
- Проверяет дубликаты перед добавлением  

### 4. Извлечение GitHub-репозиториев  
При анализе любой статьи агент автоматически:  
- Сканирует текст на наличие ссылок на GitHub  
- Загружает README через curl  
- Извлекает информацию: архитектуру, алгоритмы, авторов, лицензию, зависимости  
- Создаёт отдельную wiki-страницу с структурой:  
  - Обзор  
  - Архитектура  
  - Ключевые алгоритмы  
  - Установка  
  - Поддерживаемые датасеты
  - Связанные статьи  
  - Лицензия  
- Устанавливает двусторонние связи между статьёй и репозиторием  

### 5. Рейтинг и обратная связь
Каждая статья в pm-wiki имеет рейтинг. Пользователь может поставить лайк или дизлайк ответу агента, и агент обновит рейтинг использованных статей:

| Рейтинг | Интерпретация |
|---------|---------------|
| ≥ 5 | ★★★★★ — надёжное знание, приоритетное |
| 3–4 | ★★★★☆ — хорошее знание |
| 1–2 | ★★★☆☆ — неплохое |
| 0 | ★★☆☆☆ — нейтральное |
| < 0 | ★☆☆☆☆ — сомнительное |
