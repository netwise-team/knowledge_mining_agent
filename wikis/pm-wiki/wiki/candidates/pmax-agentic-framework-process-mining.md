# PMAx: Агентный фреймворк для AI-управляемого процессного майнинга

**PMAx** — это автономный агентный фреймворк, разработанный для демократизации [[process-mining-handbook|процессного майнинга]] посредством использования больших языковых моделей (LLM). Фреймворк создан исследователями Антоном Антоновым, Хумамом Коурани, Алессандро Берти, Гюнамом Парком и Вилом М. П. ван дер Аалстом (Институт Фраунгофера FIT и Университет РВТХ Ахен). Статья подана на рассмотрение в трек демонстрации инструментов конференции EMMSAD 2026 (arXiv:2603.15351).^[2603.15351:14-31]

## Мотивация и проблематика

Процессный майнинг предоставляет мощные инструменты для анализа организационных рабочих процессов, однако извлечение аналитических выводов традиционно требует экспертизы в специализированных языках запросов и инструментах науки о данных. LLM открывают возможность сделать [[process-mining-handbook|процессный майнинг]] доступным для бизнес-пользователей через взаимодействие на естественном языке.^[2603.15351:29-29]

Тем не менее прямое использование LLM в качестве аналитических движков над сырыми журналами событий сопряжено с фундаментальными проблемами:

- **Детерминированное рассуждение**: LLM испытывают трудности с точными вычислениями метрик и могут генерировать галлюцинированные результаты — явление, подробно изученное в контексте [[knowledge-driven-hallucination-llm-process-modeling|knowledge-driven hallucination]].
- **Конфиденциальность данных**: передача больших и чувствительных журналов событий внешним AI-сервисам создаёт серьёзные риски для приватности.^[2603.15351:29-29]

## Архитектура PMAx

PMAx решает указанные проблемы через **приватно-сохраняющую мультиагентную архитектуру**, разделяющую вычисление и интерпретацию результатов.^[2603.15351:29-29]

### Агент Engineer (Инженер)

Агент Engineer анализирует метаданные журнала событий и автономно генерирует локальные скрипты для:

- запуска установленных алгоритмов [[process-mining-handbook|процессного майнинга]];
- вычисления точных метрик;
- создания артефактов: процессных моделей, сводных таблиц и визуализаций.

Все вычисления выполняются локально, что исключает передачу сырых данных внешним сервисам.^[2603.15351:29-29]

### Агент Analyst (Аналитик)

Агент Analyst интерпретирует инсайты и артефакты, полученные агентом Engineer, и компилирует из них комплексные аналитические отчёты на естественном языке, доступные нетехническим пользователям.^[2603.15351:29-29]

## Ключевые свойства

| Свойство | Описание |
|---|---|
| Математическая точность | Вычисления выполняются детерминированными алгоритмами, а не LLM |
| Конфиденциальность данных | Анализ производится локально без отправки журналов событий во внешние сервисы |
| Доступность | Бизнес-пользователи формулируют вопросы на естественном языке |
| Автономность | Фреймворк функционирует как виртуальный процессный аналитик |^[2603.15351:29-29]

## Связь с исследованиями по LLM и процессному майнингу

PMAx непосредственно опирается на результаты исследований по [[knowledge-driven-hallucination-llm-process-modeling|knowledge-driven hallucination в LLM]], демонстрируя, что прямое применение LLM для генерации процессных моделей или вычисления метрик ненадёжно. Вместо этого фреймворк использует LLM исключительно для интерпретации и коммуникации, тогда как аналитические вычисления делегируются проверенным алгоритмам [[process-mining-handbook|процессного майнинга]].^[2603.15351:29-29]

Авторы PMAx — те же исследователи, которые изучали [[knowledge-driven-hallucination-llm-process-modeling|knowledge-driven hallucination]] и разрабатывали алгоритмы [[partial-order-powl-process-discovery|обнаружения процессов на основе POWL]], что свидетельствует о системном подходе группы RWTH Aachen / Fraunhofer FIT к интеграции LLM в процессный майнинг.
