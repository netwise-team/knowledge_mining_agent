# gOCED: Онтологические основания объектно-ориентированных событийных данных

Статья «Time and Relations into Focus: Ontological Foundations of Object-Centric Event Data» (arXiv:2512.14425) предлагает новую метамодель объектно-ориентированных событийных данных — **gOCED** — основанную на формальной онтологии. Авторы: Hosna Hooshyar, Mattia Fumagalli, Marco Montali и Giancarlo Guizzardi. Работа опубликована на arXiv в декабре 2025 года (v2 — март 2026). ^[2512.14425:15-49]

## Мотивация

[[object-centric-distance-metric|Объектно-ориентированный процессный майнинг (OCPM)]] — новое направление [[process-mining-data-science-in-action|процессного майнинга]], в котором события связаны с несколькими объектами, а взаимодействия между объектами принципиально важны для понимания динамики процесса. Традиционные, так называемые «кейс-центричные» модели событийных данных не способны адекватно отразить эту сложность. ^[2512.14425:29-29]

Существующие модели Object-Centric Event Data (OCED) стремятся сохранить простоту при переходе от кейс-центричного подхода, однако страдают от:
- **внутренних неоднозначностей** в семантике событий и объектов;
- **недостаточной поддержки временно́го измерения** — представления моментов, интервалов и порядка событий;
- **ограниченной поддержки динамических отношений** между объектами, изменяющихся в ходе процесса. ^[2512.14425:29-29]

## Трёхшаговый подход

Авторы предлагают методологию из трёх шагов для устранения указанных недостатков.

### Шаг 1: Анализ открытых проблем существующих метамоделей OCED

На основе литературы выявляются ключевые проблемы действующих метамоделей OCED: неоднозначность интерпретации событий, неполнота представления временны́х отношений и отсутствие механизмов для описания динамически меняющихся связей между объектами. Эти проблемы иллюстрируются на репрезентативных примерах. ^[2512.14425:29-29]

### Шаг 2: Онтологическое обогащение OCED Core Model

За основу берётся **OCED Core Model** — модель, предложенная в качестве базиса для нового стандарта объектно-ориентированных событийных данных. Она обогащается путём заземления на **gUFO** — облегчённую версию фундаментальной онтологии **UFO-B**, специально разработанной для представления объектов, событий, времени и их (динамических) отношений. Результатом является новая метамодель **gOCED**. ^[2512.14425:29-29]

### Шаг 3: Верификация охвата и расширения

Показывается, что gOCED:
- **охватывает возможности существующих метамоделей**, сохраняя их простоту;
- **расширяет их** за счёт функций, необходимых для преодоления выявленных неоднозначностей и ограничений выразительности. ^[2512.14425:29-29]

## Ключевые концепции

### gUFO и UFO-B

**UFO-B** (Unified Foundational Ontology, часть B) — формальная онтология, ориентированная на представление событий, объектов и их отношений во времени. **gUFO** — её облегчённая («граф-ориентированная») версия, пригодная для практического применения в инженерии знаний и моделировании данных. ^[2512.14425:29-29]

### OCED Core Model

OCED Core Model — предложенный стандарт для объектно-ориентированных событийных данных, определяющий базовые понятия: событие, объект, атрибут, отношение. gOCED расширяет эту модель онтологически обоснованными конструктами для времени и динамических отношений. ^[2512.14425:29-29]

### gOCED

**gOCED** (gUFO-grounded Object-Centric Event Data) — новая метамодель, устраняющая семантические неоднозначности и обеспечивающая:
- точное представление **временны́х структур** (моменты, интервалы, порядок);
- поддержку **динамических отношений** между объектами, изменяющихся в ходе выполнения процесса;
- совместимость с существующими подходами к [[ocpm-from-time-series-sensor-data|извлечению объектно-ориентированных событийных данных]]. ^[2512.14425:29-29]

## Связь с другими работами

Работа вписывается в более широкий контекст развития [[streaming-object-centric-process-mining|объектно-ориентированного процессного майнинга]] и стандартизации форматов событийных данных. Онтологическое обоснование OCED дополняет практические подходы к [[ocpm-from-time-series-sensor-data|трансформации сенсорных данных в OCED]] и [[object-centric-streaming-process-discovery-liss|потоковому обнаружению объектно-ориентированных процессов]], предоставляя им формальный семантический фундамент.
