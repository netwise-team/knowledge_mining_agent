# pm4py-mdl: Модуль многомерных журналов для PM4Py

**pm4py-mdl** (Multi-Dimensional Log Module for PM4Py) — это открытый программный модуль на языке Python, разработанный Alessandro Berti (псевдоним на GitHub: Javert899) и предназначенный для работы с многомерными журналами событий в рамках экосистемы [[process-mining-handbook|процессного майнинга]]. Репозиторий доступен на GitHub по адресу `Javert899/pm4py-mdl` и распространяется под лицензией MIT. ^[pm4py-mdl:40-117]

## Назначение и концепция

Многомерные журналы (Multi-Dimensional Logs, MDL) расширяют классическую модель журнала событий, позволяя одновременно отслеживать несколько взаимосвязанных процессов и объектов в рамках единой структуры данных. Это концептуально близко к подходу [[process-scope-enrichment-ocel|объектно-ориентированных журналов событий (OCEL)]], однако реализовано в виде отдельного модуля поверх библиотеки PM4Py.

Основная идея состоит в том, что реальные бизнес-процессы редко существуют изолированно: заказ связан с накладной, накладная — с платёжным документом, и т.д. MDL позволяет моделировать такие межпроцессные зависимости явно, не сводя их к единой плоской трассе.

## Структура репозитория

Репозиторий содержит следующие ключевые компоненты:

- **`pm4pymdl/`** — основной пакет библиотеки с алгоритмами обработки многомерных журналов.
- **`example_code/`** — примеры использования модуля.
- **`example_logs/`** — образцы журналов событий для тестирования.
- **`controllers/`**, **`templates/`**, **`static/`** — компоненты веб-интерфейса (Flask-приложение).
- **`sap/`** — интеграционные скрипты для работы с данными SAP.
- **`p2p_mdl.py`**, **`vbfa_mdl.py`** — примеры построения MDL для процессов «Закупка-к-оплате» (P2P) и документооборота SAP (VBFA).
- **`BUILD_GRAPH.py`** — утилита построения графов зависимостей между процессами.
- **`Dockerfile`** — конфигурация для контейнеризации приложения. ^[pm4py-mdl:57-107]

## Интеграция с SAP

Одной из заметных особенностей pm4py-mdl является наличие специализированных скриптов для извлечения и преобразования данных из систем SAP (файлы `bkpf_parquet.py`, `vbfa_parquet.py`, `extractor.py`). Это указывает на практическую ориентацию модуля на корпоративные ERP-системы, что характерно для прикладного [[process-mining-handbook|процессного майнинга]] в крупных организациях. ^[pm4py-mdl:76-107]

## Связь с другими инструментами

pm4py-mdl строится поверх библиотеки PM4Py — основного инструментария для [[process-mining-handbook|процессного майнинга]] на Python. Концептуально модуль пересекается с направлением объектно-ориентированного процессного майнинга, представленным в [[process-scope-enrichment-ocel|обогащении объектно-ориентированных журналов событий процессными областями]], а также с подходами к многомодальной векторизации данных, реализованными в [[veco-multimodal-process-mining-library|библиотеке VeCo]].

Многомерный подход к журналированию также концептуально связан с задачами многогранного исследования процессов, рассматриваемыми в [[coordinated-projections-multi-faceted-process-exploration|координированных проекциях для многогранного исследования процессов]].
