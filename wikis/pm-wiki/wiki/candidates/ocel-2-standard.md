# OCEL 2.0: Стандарт объектно-ориентированных журналов событий

OCEL 2.0 (Object-Centric Event Log 2.0) — это формат и стандарт для объектно-ориентированных журналов событий, составляющих основу [[object-centric-distance-metric|объектно-ориентированного процессного майнинга (OCPM)]]. Стандарт представляет собой более выразительную версию по сравнению с первоначальным форматом OCEL 1.0 и позволяет проводить более глубокий анализ процессов, сохраняя при этом удобный для обмена формат данных. ^[ocel-standard.org:3-5]

## Назначение и мотивация

Объектно-ориентированные журналы событий (OCEL) служат входными данными для [[streaming-object-centric-process-mining|объектно-ориентированного процессного майнинга]], позволяя моделировать реальные процессы, в которых одно событие может быть связано с несколькими объектами разных типов. Классические журналы событий, ориентированные на единственный идентификатор случая (case ID), не способны адекватно отразить такую многообъектную природу процессов.

OCEL 2.0 устраняет ключевые ограничения первого стандарта: он поддерживает отслеживание изменений атрибутов объектов во времени и предоставляет расширенные возможности описания взаимосвязей между объектами. Стандарт применяется в таких областях, как [[process-scope-enrichment-ocel|обогащение объектно-ориентированных данных о событиях]], [[ocpm-personal-health-management|управление здоровьем на основе OCPM]], [[ocpm-collection-objects-split-tree-mining|майнинг коллекционных объектов]] и [[exoar-object-activity-recognition-ocpm|распознавание объектов и активностей]]. ^[ocel-standard.org:5-5]

## Основные элементы стандарта

### События (Events)

Событие описывает конкретное выполнение активности и является базовой единицей журнала событий. Каждое событие содержит:
- временну́ю метку (timestamp);
- ссылки на объекты, с которыми или над которыми была выполнена активность;
- значения атрибутов, специфичных для данной активности. ^[ocel-standard.org:7-7]

### Объекты (Objects)

Каждый объект — это уникально идентифицируемый экземпляр определённого типа объектов, например «order-123». Объект обладает значениями атрибутов, определённых для своего типа, причём эти значения могут изменяться в разные моменты времени. Поддержка изменений атрибутов во времени является одним из ключевых улучшений по сравнению с OCEL 1.0. ^[ocel-standard.org:9-9]

### Взаимосвязи (Relationships)

OCEL 2.0 поддерживает два вида взаимосвязей:

1. **Связь событие–объект (event-to-object relationship)**: каждое событие связано как минимум с одним объектом.
2. **Связь объект–объект (object-to-object relationship)**: два объекта могут быть связаны между собой без совместного участия в каком-либо событии.

Оба типа взаимосвязей теперь могут быть **квалифицированы** (см. ниже). ^[ocel-standard.org:11-11]

### Активности (Activities)

Активность — это тип события, которое было выполнено, например «зарегистрировать заказ клиента». Каждая активность может ссылаться на набор атрибутов, таких как место выполнения. ^[ocel-standard.org:13-13]

### Типы объектов (Object Types)

Каждый объект принадлежит определённому типу объектов. Типы объектов задают набор атрибутов, значения которых могут различаться для разных объектов одного типа, а также изменяться во времени для одного и того же объекта. ^[ocel-standard.org:15-15]

### Квалификаторы (Qualifiers)

Оба типа взаимосвязей — событие–объект и объект–объект — могут быть снабжены квалификатором, уточняющим роль объекта в событии или характер связи между объектами. Например, квалификатор позволяет указать, является ли объект исполнителем, получателем или иным участником события. Это нововведение существенно повышает семантическую выразительность журнала. ^[ocel-standard.org:17-17]

## Отличия от OCEL 1.0

| Характеристика | OCEL 1.0 | OCEL 2.0 |
|---|---|---|
| Изменения атрибутов объектов во времени | Не поддерживается | Поддерживается |
| Связи объект–объект | Отсутствуют | Присутствуют |
| Квалификаторы взаимосвязей | Отсутствуют | Присутствуют |
| Выразительность анализа | Базовая | Расширенная |

## Применение в процессном майнинге

Формат OCEL 2.0 используется как входной стандарт в широком спектре задач объектно-ориентированного процессного майнинга. В частности:
- [[streaming-object-centric-process-mining|Потоковый объектно-ориентированный процессный майнинг (SOCPM)]] опирается на объектно-ориентированные журналы событий для онлайн-обнаружения моделей процессов.
- [[process-scope-enrichment-ocel|Обогащение данных OCEL областями процессов]] расширяет стандарт явными аналитическими областями.
- [[ocpm-personal-health-management|Применение OCPM в управлении здоровьем]] предлагает расширение OCEL сенсорными данными.
- [[exoar-object-activity-recognition-ocpm|ExOAR]] извлекает структурированные объектно-ориентированные данные о событиях из неструктурированных текстов для последующего представления в формате OCEL.
