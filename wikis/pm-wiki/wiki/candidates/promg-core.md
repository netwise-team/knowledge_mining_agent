# PromG-core: Библиотека для многомерного анализа процессов на основе графов знаний событий

**PromG-core** — это открытая Python-библиотека, разработанная организацией PromG-dev, которая позволяет выполнять **многомерный анализ процессов** (multi-dimensional process analytics) с использованием многоуровневых **графов знаний событий** (Event Knowledge Graphs, EKG). Библиотека доступна на PyPI под именем `promg` и распространяется по лицензии LGPL-3.0.^[promg-core:96-116]

## Назначение и концепция

PromG-core предоставляет инструментарий для [[process-mining-handbook|майнинга процессов]], ориентированный на работу с многоуровневыми графами знаний событий. В отличие от традиционных подходов, основанных на плоских журналах событий, PromG использует граф-ориентированное представление данных, в котором события, объекты и их взаимосвязи хранятся в виде узлов и рёбер. Это позволяет проводить анализ сразу по нескольким измерениям процесса — например, по различным типам объектов, ресурсам и временны́м перспективам — в рамках единой аналитической модели.

Данный подход концептуально близок к [[streaming-object-centric-process-mining|объектно-ориентированному майнингу процессов]] и методам работы с [[process-scope-enrichment-ocel|объектно-ориентированными журналами событий (OCEL)]], поскольку также выходит за рамки классической модели «один процесс — один журнал».

## Основные характеристики

- **Установка:** библиотека устанавливается стандартным образом через менеджер пакетов Python:
  ```
  pip install promg
  ```
- **Язык реализации:** Python (100% кодовой базы).
- **Лицензия:** GNU Lesser General Public License v3.0 (LGPL-3.0).
- **Репозиторий:** [github.com/PromG-dev/promg-core](https://github.com/PromG-dev/promg-core), 654 коммита, 17 релизов (актуальная версия — v2.4.7).
- **Документация:** доступна по адресу [promg-dev.github.io/promg-core](https://promg-dev.github.io/promg-core/), включает руководство по началу работы и обучающие материалы.^[promg-core:97-116]

## Демонстрация и примеры

Авторы подготовили демонстрационное видео, в котором PromG применяется к набору данных **BPIC'17** (Business Process Intelligence Challenge 2017) — широко используемому эталонному набору данных в сообществе [[process-mining-handbook|майнинга процессов]]. Репозиторий также содержит примеры анализов, иллюстрирующие возможности библиотеки.^[promg-core:99-102]

## Граф знаний событий как основа анализа

Ключевой абстракцией PromG является **многоуровневый граф знаний событий** (multi-layered Event Knowledge Graph). В такой структуре:

- каждое событие представлено узлом графа;
- объекты, участвующие в событиях, также являются узлами;
- рёбра кодируют отношения «событие произошло с объектом», «событие предшествует событию», «объект связан с объектом» и т. д.;
- различные «слои» графа соответствуют различным перспективам или типам сущностей.

Эта модель позволяет обнаруживать паттерны взаимодействия между несколькими процессами и объектами одновременно, что делает её мощным инструментом для сценариев, аналогичных тем, которые рассматриваются в [[ocpm-from-time-series-sensor-data|объектно-ориентированном майнинге процессов из данных датчиков]] и [[deviation-desirability-assessment|проверке соответствия]].^[promg-core:96-96]

## Связанные инструменты

PromG-core занимает нишу, смежную с такими инструментами, как [[veco-multimodal-process-mining-library|VeCo]], ориентированными на векторизацию данных для задач майнинга процессов, однако фокусируется именно на граф-ориентированном хранении и анализе событийных данных.
