# Обнаружение объектно-ориентированных сетей Петри

Статья «Discovering Object-Centric Petri Nets» (arXiv:2010.02047) написана **Вилом М. П. ван дер Аалстом** и **Алессандро Берти** и опубликована как препринт в октябре 2020 года; окончательная версия вышла в журнале *Fundamenta Informaticae*. Работа вводит формализм **объектно-ориентированных сетей Петри** (object-centric Petri nets, OCPN) и соответствующий алгоритм обнаружения процессов из объектно-ориентированных журналов событий, реализованный в библиотеке [[process-mining-data-science-in-action|PM4Py]].^[2010.02047:12-24]

## Проблема единственного идентификатора случая

Традиционные методы [[process-mining-handbook|обнаружения процессов]] предполагают наличие ровно одного идентификатора случая (case identifier) на каждое событие. Этот идентификатор используется для корреляции событий, а полученная сеть Петри описывает жизненный цикл отдельного случая. Однако в реальных информационных системах одно событие может одновременно относиться к нескольким объектам разных типов — например, к заказу, позиции заказа, посылке, клиенту и продукту. Принудительное сведение такой многомерной структуры к единственному идентификатору случая порождает артефакты: дублирование событий (event replication) и расхождение поведения (behavior divergence), искажающие обнаруженную модель.^[2010.02047:22-22]

## Объектно-ориентированные журналы событий

Авторы предлагают использовать **объектно-ориентированные журналы событий** (object-centric event logs), в которых каждое событие связано с коллекцией объектов, каждый из которых имеет определённый тип. Такой формат значительно ближе к структуре данных реальных ERP- и CRM-систем, где таблицы базы данных естественным образом хранят отношения «многие ко многим» между событиями и объектами.^[2010.02047:22-22] Объектно-ориентированный журнал событий является непосредственным предшественником формата [[object-centric-streaming-process-discovery-liss|OCEL]], ставшего стандартом де-факто в объектно-ориентированном процессном майнинге.

## Формализм объектно-ориентированных сетей Петри

Объектно-ориентированная сеть Петри расширяет классическую сеть Петри следующим образом:

- **Места** соответствуют типам объектов; каждое место хранит мультимножество объектов соответствующего типа.
- **Переходы** могут потреблять и производить коллекции объектов разных типов одновременно, отражая реальные взаимодействия между объектами.
- **Дуги** аннотированы переменными, связанными с конкретными типами объектов, что позволяет задавать гибкие отношения потребления и производства.

Такая структура позволяет визуализировать сложные взаимосвязи между объектами разных типов в рамках единой целостной модели процесса.^[2010.02047:22-22]

## Алгоритм обнаружения

Предложенный алгоритм обнаружения работает следующим образом:

1. **Извлечение плоских журналов**: для каждого типа объектов из объектно-ориентированного журнала извлекается «плоский» журнал событий путём проецирования на данный тип.
2. **Обнаружение per-type**: к каждому плоскому журналу применяется стандартный алгоритм обнаружения (например, алгоритм Альфа или Inductive Miner), что даёт отдельную сеть Петри для каждого типа объектов.
3. **Интеграция**: полученные сети объединяются путём слияния переходов, соответствующих одним и тем же активностям, в единую объектно-ориентированную сеть Петри.
4. **Детализация**: итоговая модель допускает «детализацию» (drill-down) до конкретной точки зрения путём проецирования на выбранный тип объектов.

Алгоритм реализован в библиотеке **PM4Py** и доступен для практического применения.^[2010.02047:22-22]

## Значение и влияние

Данная работа заложила теоретический фундамент объектно-ориентированного процессного майнинга. Введённые концепции — объектно-ориентированный журнал событий, объектно-ориентированная сеть Петри и алгоритм обнаружения через проецирование — стали отправной точкой для последующих исследований, включая разработку стандарта OCEL, методов потоковой обработки объектно-ориентированных данных ([[object-centric-streaming-process-discovery-liss]]) и алгоритмов обнаружения на основе частичных порядков ([[partial-order-powl-process-discovery]]). Работа также включена в учебные материалы [[process-mining-handbook|летней школы по процессному майнингу]] как ключевой источник по объектно-ориентированному подходу.

## Реализации и связанные инструменты (GitHub)

Алгоритм OCPN реализован в нескольких открытых проектах:

- [[opera-object-centric-performance-analysis|OPerA]] (github.com/gyunamister/OPerA) — интерактивный инструмент для объектно-ориентированного анализа производительности; использует алгоритм обнаружения OCPN из данной статьи (Inductive Miner Directly-Follows) как ключевой компонент.
- [[ocpa]] (github.com/ocpm/ocpa) — Python-библиотека для объектно-ориентированного процессного майнинга (Jan Niklas Adams, Gyunam Park, Wil van der Aalst); реализует обнаружение OCPN, проверку соответствия, анализ производительности и предиктивный мониторинг.
