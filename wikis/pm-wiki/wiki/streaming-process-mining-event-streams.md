# Streaming Process Mining and Intentional Event Streams

Streaming Process Mining (SPM) extends traditional [[process-mining-data-science-in-action|process mining]] to handle real-time, continuous event data rather than static, retrospectively collected event logs. The shift toward IoT-enabled and sensor-driven systems — characteristic of Industry 4.0 and distributed manufacturing environments — has made SPM increasingly important for operational monitoring and early fault detection.

## Motivation: From Static Logs to Event Streams

Traditional process mining assumes event logs are complete, static, chronologically ordered by case, and collected retrospectively. These assumptions support post-mortem analysis but are ill-suited to real-time environments where:

- Events arrive asynchronously and possibly **out of order**
- Cases may have **indefinite lifespans** with no clear completion signal
- **Concurrent activities** occur across multiple process instances simultaneously
- **Concept drifts** arise when process behavior shifts (e.g., a new machine is introduced or a production line is reconfigured)

SPM aims to process events on the fly, enabling early detection of issues rather than retrospective explanation. However, existing SPM evaluation practices rely on *streamified* event logs — static logs replayed incrementally — or synthetic logs generated by tools such as CPN Tools. These approximations lack core properties of realistic event streams and undermine the development and benchmarking of SPM algorithms.^[Maldonado et al - Know Your Streams On the Conceptualization, Characterization and Generation of Intentional Event Streams.pdf:49-62]

## Know Your Streams: A Conceptual Framework

Maldonado, Imenkamp, Reiter, Seidl, Hasselbring, Werner, and Koschmider (TU Munich, University of Bayreuth, Christian-Albrechts-University Kiel, LMU Munich) introduced the *Know Your Streams* framework, accepted at the ICPM 2025 Workshops (Springer LNBIP). The framework makes three contributions:

1. **Identification of ES characteristics** from the general data stream community not yet reflected in SPM
2. **Extended event stream formalization** capturing realistic stream properties
3. **Stream of Intent**, a prototype generator for intentional event streams

^[Maldonado et al - Know Your Streams On the Conceptualization, Characterization and Generation of Intentional Event Streams.pdf:76-80]

## Extended Event Stream Formalization

### Interval-Based Events

Classical process mining represents an event as an atomic tuple `e ∈ E : c × a × t` (case id, activity, timestamp). This is insufficient for activities with duration. The framework introduces paired start/end events:

- `es = (c, a, t, "start") ∈ Es`
- `ee = (c, a, t, "end") ∈ Ee`

A mapping function `µ : Es → Ee` links each start event to its corresponding end event, enforcing same case id, same activity, and `πt(es) ≤ πt(ee)`. This enables formal definition of **concurrency** at a given timestamp `t*` as the count of active start events whose intervals contain `t*`.

### Temporal Reality and Out-of-Order Events

The event tuple is further extended to `e = (c, a, t, type, t_arrival)`, distinguishing:
- `t`: the event timestamp (when the event occurred in the real-world process)
- `t_arrival`: the arrival timestamp (when the event was received by the analysis system)

The stream is ordered by arrival time. An **out-of-order event** is one whose event timestamp is earlier than that of a preceding event in the arrival sequence. The displacement `πt(ei) − πt(ej)` quantifies the degree of disorder.

### Dependencies

The history of a case `c` at time `t`, denoted `H(c, t)`, is the ordered sequence of all events for that case up to time `t`. A transition function `δ` maps case history to the next activity:

- **Temporal dependencies**: time elapsed between events is a direct consequence of `δ` operating on event history
- **Long-term dependencies**: `δ` is sensitive to events early in the history (distant past influences current activities)
- **Non-linear dependencies**: `δ` models complex branching, parallelism, or context-aware decisions rather than simple sequential logic

### Hierarchical Structures and Fractal Behavior

Real-world processes can be nested or trigger sub-processes. A case containment relation `cj ≺ ci` signifies that `cj` is a sub-process of `ci`, forming a directed acyclic graph over cases. **Self-similarity** (fractal behavior) exists when the process model of a sub-case is structurally related to its parent by a transformation `T` (e.g., temporal scaling or structural simplification).

## Stream of Intent: Prototype Generator

Stream of Intent is a pipeline for generating synthetic event streams with intentionally predefined, controllable characteristics. It builds on:

- **DEF (Distributed Event Factory)**: a simulation engine for generating event streams from distributed data sources
- **GEDI (Generating Event Data with Intentional Features)**: a feature-driven optimization framework for [[data-quality-sensitivity-process-discovery|benchmarking process mining]] algorithms

### Pipeline Steps

1. **Specify stream features and targets**: the user defines target values for features such as event frequency, concurrency patterns, or dependency types
2. **Generate process model**: a stochastic process tree is generated using methods such as PTandLogGenerator (PTLG)
3. **Process tree to Markov chain conversion**: the process tree is converted to a first-order Markov chain by generating a sample log, counting activity transition frequencies, and normalizing to a transition matrix
4. **Generate stream (DEF simulation)**: DEF simulates an event stream from the Markov chain, accounting for concurrent events, event lifecycles, and durations
5. **Feature extraction and evaluation**: the simulated stream is batched using a tumbling window; predefined features are extracted and compared to targets
6. **Bayesian optimization**: generator parameters are iteratively adjusted until target feature specifications are met or a maximum number of attempts is reached

The output is a static definition file compatible with DEF, replayable with optimized features. DEF's ability to combine multiple definitions also enables simulation of **concept drift** (gradual or sudden transitions between feature sets).

## Experimental Evaluation

Two evaluation questions were investigated across 250 experiments (all pairwise combinations of five features × five target values):

- **(E1) Feasibility**: Stream of Intent closely matches predefined target feature values. Some feature combinations are harder to achieve (e.g., Non-Linear Dependency combined with high Temporal or Long-Term Dependency targets; Fractal Behavior requires Non-Linear Dependencies).
- **(E2) Representativeness**: Principal component analysis comparing generated streams to benchmark event logs commonly used in SPM evaluation shows that existing event logs do not cover most characteristics of realistic event streams — no benchmark log achieves high values on the second principal component, meaning key stream features are simply absent from standard benchmarks.

Experiments with established SPM algorithms (streaming heuristics miner and others) confirm that current algorithms require extensive manual preprocessing to handle realistic stream characteristics, and that realistic streams produce significantly more silent transitions and loops than streamified logs.^[Maldonado et al - Know Your Streams On the Conceptualization, Characterization and Generation of Intentional Event Streams.pdf:85-104]

## Implications for SPM Algorithm Development

The framework highlights that:

- Existing SPM algorithms are designed around assumptions inherited from classical process mining and do not natively handle realistic stream characteristics
- Extensive preprocessing to convert streams into an "offline" format is often infeasible in real-time settings
- Novel streaming discovery and [[process-mining-handbook|conformance checking]] approaches are needed that can natively handle concurrency, out-of-order events, long-term dependencies, and fractal structures

^[Maldonado et al - Know Your Streams On the Conceptualization, Characterization and Generation of Intentional Event Streams.pdf:86-93]

## Relationship to Other Work

Stream of Intent connects to the [[veco-multimodal-process-mining-library|VeCo]] library ecosystem (Agnes Koschmider, University of Bayreuth) and to broader [[process-mining-handbook|process mining]] benchmarking concerns. The absence of case identifiers in real-world streams also motivates online correlation techniques as a prerequisite for SPM.
